## Project Overview

KS TalentBridge is an internal AI-powered staffing intelligence and proposal support system for KS Engineers, a ~350-person AEC firm headquartered in Newark, NJ.

**Primary objective:**

Convert unstructured resumes and project experience data into a searchable knowledge system that allows proposal teams to rapidly identify qualified staff.

**Typical questions the system must answer:**

- Who holds a PA Professional Engineer license?
- Who has SEPTA project experience?
- Who completed NHI 130055 bridge inspection training?
- Who holds NICET certifications?
- Who has Fracture Critical bridge inspection experience?

These results are used in proposal submissions and marketing materials, so accuracy and evidence are critical.

---

## System Design Philosophy

TalentBridge prioritizes:

- High recall over perfect precision

**Acceptable workflow:**
- AI returns 15–25 candidates
- Human reviewer narrows results

The system must not miss qualified staff.

False positives are acceptable if clearly indicated and verifiable.

## System Architecture

The system uses a three-layer knowledge retrieval model.

```
User Query
   │
   ▼
ChatGPT Teams (SharePoint connector)
   │
   ▼
Layer 1 — Structured Index
KSE_Master_Index.csv
   │
   ▼
Layer 2 — Full Text Corpus
KSE_FullText_KB.jsonl
   │
   ▼
Layer 3 — Source Documents
SharePoint: Resume 08152025
```

This architecture ensures:
- deterministic filtering
- semantic recall
- document evidence verification

## AI Query Protocol

The system prompt instructs the model to follow this retrieval process.

### STEP 0 — Structured Index Query

**Search:** `KSE_Master_Index.csv`

**Use for:** licenses, certifications, training, agency flags, roles, count queries

This is the deterministic retrieval layer.

### STEP 1 — Full Text Recall

If structured index is insufficient:

**Search:** `KSE_FullText_KB.jsonl`

**Use for:** experience discovery, agency mentions, training references, rare phrases, context extraction

### STEP 2 — Document Verification

Open the resume file from `Resume 08152025/` and extract supporting evidence.

---

## AI Behavior Rules

When answering queries the system must:

1. Prioritize recall over aggressive filtering
2. Include candidates when evidence is uncertain
3. Never invent certifications or licenses
4. Provide resume evidence whenever possible
5. Return structured candidate lists

**Preferred response format:**
- Candidate Name
- Reason for inclusion
- Evidence snippet (optional)
